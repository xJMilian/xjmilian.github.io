  • The CompTIA A+ certification covers more topics than many industry certifications
  • One of the most sought-after certifications by aspiring technologists and employers
  • Requires completion of two exams: 220-1201 (Core 1) and 220-1202 (Core 2)
  • This book provides notes for the 220-1201 Core 1 exam

1.1
Laptop Hardware
Laptop Hardware
  • Engineered to precise specifications — a challenging repair
  • Some laptops are easier to fix than others — an ongoing learning process
  • Details will vary between laptop manufacturers
Laptop Batteries
  • Power source when unplugged — may require a teardown to replace
  • Lithium-Ion (Li-ion) and Lithium-Ion polymer (LiPo) are common
  • No "memory effect"
  • Charging the battery diminishes capacity over time
  • Different form factor for each laptop — technology is constantly changing
Laptop Keyboard
  • Most-used component — can be easy to replace
  • A few screws and a single ribbon cable
  • Or connect an external USB keyboard as an alternative
  • Key cap removal is delicate — check manufacturer's instructions
Laptop Memory
  • SO-DIMM — Small Outline Dual In-line Memory Module
  • Used in laptops and mobile devices
  • Often easy to install and replace
  • Some laptop memory is soldered to the system board — no upgrade available
  • Requires a full system board replacement
Laptop Storage
  • Magnetic disk — traditional spinning drive platters, 2.5" form factor
  • SSD (Solid-state drive) — all memory, no moving parts, silent, fast
  • 2.5" form factor
  • M.2 — smaller form factor, no SATA data or power cables, easy to install
Migrating HDD to SSD
  • An impressive upgrade — almost like getting a new laptop
  • Option 1: Install OS on SSD, move user documents, reinstall applications
  • Option 2: Image/clone the HDD — no OS installation required
  • Imaging software needed — sometimes included with the SSD
  • Drive-to-drive image: image directly from one drive to the other
1.1
Wireless, Biometrics & NFC
802.11 Wireless and Bluetooth
  • 802.11 — Local Area Network (LAN), high speed, internet access
  • Bluetooth — Personal Area Network (PAN), short range
  • Connect peripherals and nearby devices
  • Multiple antennas — WiFi main/aux and Bluetooth
  • Antenna wires wrap around the laptop screen (up high for better signal)
Biometrics
  • Sign in with fingerprint reader or face recognition
  • Something you ARE
  • Requires additional configuration in the OS
  • Hardware required for most options
  • Relatively secure — faces and fingerprints are quite unique
Near-Field Communication (NFC)
  • Short-distance networking — 4 centimeters or less
  • Data transfers or authentication
  • Common on mobile phones and smart watches
  • Payment method on your wrist
  • Use for authentication without typing a password
  • Hospital workstations, warehouses, manufacturing
Microphone
  • Built-in to the laptop display — useful for video calls
  • Not good for non-casual use — analog or USB microphones can help

Connecting Mobile Devices
USB Connections
  • Universal Serial Bus — high-speed wired communication
  • Micro-USB — a smaller USB connection, common worldwide
  • Older devices may use Mini-USB — slightly larger
USB-C
  • 24-pin double-sided USB connector — used for hosts and devices
  • Acts as USB 2.0/3.0/3.1/4 — different signals, same connector
  • Can transmit: DisplayPort, HDMI, Thunderbolt
Lightning (Apple)
  • Apple proprietary — 8-pin digital signals
  • Used on iPhone and iPad devices
  • Advantages over Micro-USB:
  • Higher power output for phones and tablets
  • Can be inserted either way
  • Simpler design
NFC & Bluetooth
  • NFC — Near Field Communication, send small amounts of data wirelessly
  • Payment systems, transportation, identity "card"
  • Short range with encryption support
  • Bluetooth — high speed over short distances (PAN)
  • Tethering, headsets, health monitors, smartwatches
Hotspot / Tethering
  • Turn your phone into a WiFi hotspot — your own personal wireless router
  • Extends the cellular data network to all your devices
  • Dependent on phone type and provider — may require additional charges

Mobile Device Accessories
Stylus
  • Digital stylus — a more advanced writing tool
  • Communicates directly to the device — pressure sensitivity, programmable buttons
  • Must be compatible with the tablet (e.g., Apple iPad uses Apple Pencil)
Headsets
  • Hands-free audio — earphones and microphone
  • Wired: USB (laptops), 3.5mm TRRS connector, Lightning port
  • Wireless: Bluetooth headsets
Speakers & Camera
  • Mobile audio — battery powered, Bluetooth link, stereo sound
  • Camera/Webcam — internal or external
  • Commonly external on desktops, internal on laptops/tablets/phones
  • Real-time video communication
Docking Station & Port Replicator

  • Port replicator — similar to docking station but no expansion card option
  • Usually connects using USB
Trackpad & Drawing Pad

  • External options: battery powered, Bluetooth connected
  • Drag, tap, multi-finger input for right-clicking, zooming, window control
  • Drawing pad — use active stylus with external digitizer, very precise input
1.3
Mobile Device Networks
Cellular Networks
  • Separate land into "cells" — antenna covers each cell with certain frequencies
  • Enable/disable voice and/or data — airplane mode disables all cellular
3G Technology
  • 3rd Generation — introduced in 1998
  • Usually several megabits per second
  • Enabled: GPS, mobile television, video on demand, video conferencing
4G and LTE
  • Long Term Evolution — based on GSM and EDGE
  • LTE: download rates of 150 Mbit/s
  • LTE Advanced (LTE-A): download rates of 300 Mbit/s
5G
  • Fifth generation — launched worldwide in 2020
  • Eventually 10 gigabits per second; currently 100–900 Mbit/s
  • Significant IoT impact — larger data transfers, faster monitoring
Wi-Fi & Hotspot
  • Connect to a local 802.11 network — email and internet access
  • May support voice: Wi-Fi calling (requires carrier support)
  • Hotspot: turn phone into wireless router, extend cellular to all devices
SIM Cards
  • Unique identifier for cellular devices — often a tiny physical card
  • Contains: SIM ID, phone number, carrier info, storage for contacts/messages
  • eSIM — embedded SIM inside the device, cannot be physically removed
  • Many devices support multiple SIMs
Bluetooth Pairing
  • Built-in security — use or verify a PIN
  • Pair once — future connections automatic
  • Process: Enable Bluetooth → Set discoverable → Select device → Enter/confirm PIN → Test
GPS
  • Created by the U.S. Department of Defense — 30+ satellites in orbit
  • Need to see at least 4 satellites
  • Determines: longitude, latitude, altitude
  • Mobile location services: GPS, WiFi, and cellular towers

Mobile Device Management
MDM (Mobile Device Management)
  • Manage company-owned and user-owned mobile devices
  • Centralized management — set policies on apps, data, camera
  • Control the remote device or a "partition"
  • Force screen locks and PINs
BYOD / COPE / CYOD
  • BYOD — Bring Your Own Device: employee owns the device
  • Difficult to secure — both home and work use
  • COPE — Corporate Owned, Personally Enabled
  • Company buys, used for both corporate and personal; full control
  • CYOD — Choose Your Own Device: similar to COPE but user chooses
MDM Policy Enforcement
  • Corporate email configuration — MDM configures automatically
  • Two-factor authentication — biometrics, pseudo-random auth app
  • Corporate applications — allow or restrict app installation
Mobile Device Synchronization
  • Email — corporate configurations vary
  • Data synchronization — based on data rates and speeds
  • Calendar and Contacts synchronization
  • Data caps — may have significant financial consequences
  • Verify restrictions with wireless carrier — get it in writing
2.1
Introduction to IP
TCP vs UDP
  • Both transported inside IP — OSI Layer 4 (Transport Layer)
  • Multiplexing — use many different applications at the same time
TCP — Transmission Control Protocol
  • Connection-oriented — formal connection setup and close
  • "Reliable" delivery — recovery from errors, manages out-of-order messages
  • Flow control — receiver manages how much data is sent
  • Used by: HTTPS, SSH
UDP — User Datagram Protocol
  • Connectionless — no formal open/close to the connection
  • "Unreliable" delivery — no error recovery, no retransmissions
  • No flow control — sender determines amount transmitted
  • Used by: real-time communication, DHCP, TFTP
Ports
  • IPv4 sockets: Server IP + protocol + server port / Client IP + protocol + client port
  • Non-ephemeral ports (permanent): 0–1,023 — usually servers/services
  • Ephemeral ports (temporary): 1,024–65,535 — determined by the client
  • TCP port numbers are NOT the same as UDP port numbers

2.2
Wireless Network Technologies
802.11 Standards (WiFi Generations)
  • IEEE 802.11 committee — universal standards
  • 802.11ac = Wi-Fi 5
  • 802.11ax = Wi-Fi 6 and Wi-Fi 6E
  • 802.11be = Wi-Fi 7
  • Frequencies: 2.4 GHz, 5 GHz, and 6 GHz (sometimes combined)
  • Channels — groups of frequencies, numbered by IEEE
  • Bandwidth: 20 MHz, 40 MHz, 80 MHz, 160 MHz, etc.
Bluetooth
  • Removes the wires — headsets, speakers, keyboards/mice
  • Uses the 2.4 GHz range (Unlicensed ISM band, same as 802.11)
  • Short range — most consumer devices operate to about 10 meters
RFID (Radio-Frequency Identification)
  • Access badges, inventory/assembly line tracking, pet identification
  • Radar technology — RF energy transmitted to tag, tag ID transmitted back
  • Bidirectional communication — some tags are active/powered
NFC (Near Field Communication)
  • Two-way wireless — builds on RFID (which is mostly one-way)
  • Payment systems — major credit cards, online wallets
  • Bootstrap for other wireless — NFC helps with Bluetooth pairing
  • Access token, identity "card" — short range with encryption support

2.3
Network Services
DNS Server
  • Convert names to IP addresses (and vice versa)
  • Distributed naming system — load balanced across many servers
  • Usually managed by ISP or IT department — a critical resource
DHCP Server
  • Dynamic Host Configuration Protocol — automatic IP address configuration
  • Available on most home routers
  • Enterprise DHCP: redundant, usually running on central servers
File Share
  • Centralized storage of documents, spreadsheets, videos, pictures
  • Standard file management: SMB, Apple Filing Protocol (AFP)
  • Front-end hides the protocol — copy, delete, rename, etc.
Print Server
  • Connect a printer to the network — provide printing services
  • May be software in a computer or built-in to the printer
  • Protocols: SMB, IPP (Internet Printing Protocol), LPD (Line Printer Daemon)
Mail Server
  • Store incoming mail, send outgoing mail
  • Usually managed by ISP or IT department — complex requirements
  • Usually one of the most important services (24x7 support)
Syslog / Web / Auth Servers
  • Syslog — standard for message logging, diverse systems, consolidated log
  • Web server — responds to browser requests using HTTP/HTTPS
  • Pages built with HTML/HTML5, static or dynamic
  • Authentication server — centralized login authentication, always enterprise service
Load Balancers & Proxy Servers
  • Load balancer — distribute load across multiple servers, invisible to end-user
  • Fault tolerance: server outages have no effect
  • Proxy server — intermediate server (client → proxy → actual request → client)
  • Features: access control, caching, URL filtering, content scanning
NTP / Spam Gateway / All-in-One
  • NTP — listens on udp/123, provides time synchronization
  • Critical for encryption, logins, backups, log timestamps
  • Spam gateway — stops unsolicited email before it reaches the user
  • All-in-one appliance — UTM with firewall, URL filter, malware scan, VPN endpoint
SCADA / ICS & IoT
  • SCADA — Supervisory Control and Data Acquisition
  • Large-scale, multi-site Industrial Control Systems
  • Power generation, refining, manufacturing — requires extensive segmentation
  • IoT — refrigerators, smart speakers, thermostats, doorbells
  • May require a segmented network to limit security breaches
2.4
DNS Configuration
Domain Name System
  • Translates human-readable names into computer-readable IP addresses
  • Hierarchical and distributed — 13 root server clusters (1,000+ actual servers)
  • Hundreds of generic TLDs (.com, .org, .net) and 275+ country TLDs (.us, .ca, .uk)
DNS Record Types
  • A — maps hostname to IPv4 address
  • AAAA — maps hostname to IPv6 address
  • CNAME — canonical name (alias to another hostname)
  • MX — mail exchanger record (hostname of mail server)
  • TXT — human-readable text; used for verification and email security
Email Security Records
  • DKIM — Domain Keys Identified Mail: digitally sign outgoing mail
  • Public key is in the DKIM TXT record, validated by mail servers
  • SPF — Sender Policy Framework: list of authorized sending servers
  • Prevents mail spoofing
  • DMARC — Domain-based Message Authentication, Reporting, and Conformance
  • Extension of SPF and DKIM — defines policy for unauthenticated emails
  • Accept all, send to spam, or reject — compliance reports available
DNS Lookup Example
$ dig www.professormesser.com ;; ANSWER SECTION: www.professormesser.com. 300 IN A 104.22.72.108 www.professormesser.com. 300 IN A 172.67.41.114 www.professormesser.com. 300 IN A 104.22.73.108 ;; Query time: 42 msec ;; SERVER: 9.9.9.9
2.4
DHCP
DHCP Overview
  • Dynamic Host Configuration Protocol — released in 1997
  • Provides automatic: IP address, subnet mask, gateway, DNS, NTP servers
  • Scales to thousands of systems with hands-off IP configuration
DORA Process (DHCP Leases)
  • Discover — client broadcasts to find a DHCP server
  • Offer — DHCP server sends an IP offer
  • Request — client locks in the offer
  • Acknowledge — DHCP server confirms
DHCP Scopes
  • IP address range (with excluded addresses)
  • Subnet mask, lease durations
  • Other scope options: DNS server, default gateway, VOIP servers
  • DHCP pools — each subnet has its own scope (e.g., 192.168.1.0/24)
Address Reservation
  • Administratively configured table of MAC addresses
  • Each MAC address has a matching reserved IP address
  • Also called: Static DHCP Assignment, Static DHCP, IP Reservation
2.4
VLANs and VPNs
LAN vs VLAN
  • LAN — Local Area Network: group of devices in the same broadcast domain
  • VLAN — Virtual LAN: same broadcast domain, separated LOGICALLY instead of physically
  • Configured on managed switches using 802.1Q
VPN Types
  • VPN — Virtual Private Network: encrypted data traversing a public network
  • Concentrator — encryption/decryption access device, often integrated into a firewall
  • Client-to-Site — on-demand access from a remote device via software
  • Some software configured as always-on
  • Site-to-Site — always-on, firewalls act as VPN concentrators

2.5
Network Devices
Routers
  • Routes traffic between IP subnets — Layer 3 device
  • Forwarding decisions based on IP address
  • Often connects diverse network types — LAN, WAN, copper, fiber
Switches
  • Bridging done in hardware via ASIC
  • Forwards traffic based on data link (MAC) address
  • Core of enterprise network — may provide PoE
  • Multilayer switch — includes routing functionality
Unmanaged vs Managed Switches
  • Unmanaged — plug and play, no VLANs, no management protocols, low cost
  • Managed — VLAN support (802.1Q), traffic prioritization, STP redundancy
  • Port mirroring, SNMP external management
Access Points
  • NOT a wireless router — a wireless router is router + AP in one device
  • An access point is a bridge — extends wired network onto wireless
  • Forwarding decisions based on MAC address
PoE (Power over Ethernet)
  • Power provided on an Ethernet cable — one wire for both network and electricity
  • Used for: phones, cameras, wireless access points
  • PoE — 15.4W / 350mA max (802.3)
  • PoE+ — 25.5W / 600mA max (802.3)
  • PoE++ — 51W (Type 3), 71.3W (Type 4), 960mA max
Modems & ONT
  • Cable modem — DOCSIS standard, up to 1 Gbit/s, data/voice/video
  • DSL modem — uses telephone lines, asymmetric, ~10,000 ft limit from CO
  • ONT — Optical Network Terminal, fiber to premises
  • Connects ISP fiber to copper, demarcation point
Firewalls
  • Filters traffic by port number (OSI Layer 4)
  • Some filter based on application layer
  • Can encrypt traffic into/out of network
  • Can proxy traffic — common security technique
  • Usually sits on ingress/egress of the network
NIC & Patch Panels
  • NIC — fundamental network device, every device has one
  • Contains MAC address — unique hardware designation
  • Often built-in to motherboard or added as expansion card
  • Patch Panel — combination of punch-down blocks and RJ-45 connectors
  • Runs from desks punched down once; easy to recable patch panel to switch
2.6
IPv4 and IPv6
IPv4 Addressing
  • Internet Protocol version 4 — OSI Layer 3 address
  • 32-bit address — 4 bytes (4 octets), ~4.29 billion addresses
  • Format: 192.168.1.131 (dotted decimal notation)
  • Public IPv4 — each address on Internet is unique
  • Private IP ranges (RFC 1918) — not Internet-routable, use NAT
Private IP Ranges (RFC 1918)
Range CIDR
10.0.0.0 – 10.255.255.255 10.0.0.0/8
172.16.0.0 – 172.31.255.255 172.16.0.0/12
192.168.0.0 – 192.168.255.255 192.168.0.0/16
IPv6 Addressing
  • 128-bit address — 340 undecillion addresses
  • Format: fe80::5d18:652:cffd:8f52 (colon-separated hexadecimal)
  • First 64 bits — network prefix (/64)
  • Last 64 bits — host network address
  • Your DNS is very important with IPv6!
2.6
Assigning IP Addresses
IPv4 Configuration Components
  • IP Address — every device needs a unique IP address
  • Subnet mask — used by local device to determine its subnet (e.g., 255.255.255.0)
  • Default gateway — router for communicating outside local subnet
  • Must be an IP address on the local subnet
Static vs Dynamic IP
  • Static — manually configured, doesn't change without manual edit
  • Difficult to manage at scale — try to avoid
  • Dynamic (DHCP) — assigned from a pool, occasionally changes
  • DHCP Reservation — best of both: static address managed at DHCP server
  • Associate specific MAC address with an IP address
APIPA (Automatic Private IP Addressing)
  • Link-local address — no forwarding by routers
  • Range: 169.254.0.0 – 169.254.255.255
  • Functional block: 169.254.1.0 – 169.254.254.255
  • Randomly assigned — uses ARP to confirm address is not in use
  • Assigned automatically when no DHCP server is found
2.7
Internet Connection Types
Satellite
  • Non-terrestrial communication — high cost relative to terrestrial
  • ~100 Mbit/s down, 5 Mbit/s up common; high latency (250ms+)
  • Starlink: advertises 25–60 ms latency
  • Issues: line of sight, rain fade
Fiber
  • High speed via frequencies of light
  • Higher installation cost — equipment costly, difficult to repair
  • Large WAN core installation: SONET, wavelength division multiplexing
  • Slowly approaching homes and businesses
Cable (DOCSIS)
  • Broadband — multiple frequencies, different traffic types
  • DOCSIS standard — 50 Mbit/s to 1,000+ Mbit/s
  • Multiple services: data, voice, video
DSL
  • Uses telephone lines — asymmetric (download faster than upload)
  • ~200 Mbit/s down / 20 Mbit/s up common
  • ~10,000 ft limit from central office (CO)
Cellular & WISP
  • Cellular — tethering or mobile hotspot, separate land into "cells"
  • WISP — Wireless ISP: terrestrial using wireless, serves rural/remote areas
  • Meshed 802.11, 5G home internet, proprietary wireless
  • Speeds: ~10 to 1,000 Mbit/s; needs outdoor antenna
2.7
Network Types
LAN & WAN
  • LAN — Local Area Network: building or group of buildings, high-speed (Ethernet, 802.11)
  • WAN — Wide Area Network: spanning the globe, connects LANs, much slower
  • Technologies: Point-to-point serial, MPLS, terrestrial and non-terrestrial
PAN & MAN
  • PAN — Personal Area Network: Bluetooth, IR, NFC, your own private network
  • Automobile audio, wireless headset, health monitoring
  • MAN — Metropolitan Area Network: city-scale, larger than LAN, smaller than WAN
  • Common: Metro Ethernet, often government-owned (right-of-way)
SAN & WLAN
  • SAN — Storage Area Network: block-level access, looks/feels like local storage
  • Requires lots of bandwidth — isolated network, high-speed tech
  • WLAN — Wireless LAN: 802.11 technologies, mobility within a building
  • Expand coverage with additional access points
2.8
Network Tools
Cable Crimper
  • "Pinch" the connector onto a wire — coaxial, twisted pair, fiber
  • Connects modular connector to Ethernet cable
  • Metal prongs pushed through insulation
  • Best practice: good crimper, correct connectors, practice!
Punch-Down Tool
  • "Punch" a wire into a wiring block — 66 block, 110 block
  • Trims the wires during the punch
  • Best practice: cable management, maintain twists, document everything
Tone Generator & WiFi Analyzer
  • Tone generator — puts an analog sound on the wire for wire tracing
  • Inductive probe — doesn't need to touch copper; hear tone through speaker
  • WiFi analyzer — identify errors and interference, validate antenna placement
Cable Testers & Loopback Plugs
  • Cable tester — continuity test, identify missing pins or crossed wires
  • Not usually used for frequency testing (crosstalk, signal loss)
  • Loopback plugs — test physical ports, fool applications
  • Serial, network (Ethernet, T1, Fiber) — NOT crossover cables
Taps and Port Mirrors
  • Intercept network traffic — send copy to a packet capture device
  • Physical tap — disconnect link, put tap in middle (active or passive)
  • Port mirror — SPAN (Switched Port ANalyzer), software-based tap
  • Limited functionality but can work well in a pinch
3.1
Display Types
LCD (Liquid Crystal Display)
  • Light shines through liquid crystals
  • Advantages: lightweight, relatively low power and inexpensive
  • Disadvantages: black levels challenging, requires separate backlight, lights difficult to replace
  • TN — Twisted Nematic: fast response (gaming!), poor viewing angles
  • IPS — In Plane Switching: excellent color, can be more expensive
  • VA — Vertical Alignment: good color, often slower response than TN
OLED & Mini LED
  • OLED — Organic Light Emitting Diode: no backlight, organic compound provides light
  • Thinner, lighter, flexible — tablets, phones, smart watches
  • Very accurate color, tends to be more expensive than LCD
  • Mini LED — same backlight tech as LED-LCD but much smaller LEDs
  • Each LED can be individually enabled/disabled and varied in color
  • Much better control over dark areas — deeper blacks, better color
Touchscreen & Digitizer
  • Touchscreen — merge laptop and tablet input, digitizer responds to touch
  • No keyboard required (but often still available)
  • Digitizer — pen-like device as input (stylus), useful for graphical input
  • Used commonly on laptop/tablets or hybrid devices
Backlight and Inverter
  • LCD displays need a backlight — fluorescent lamp or LED lights
  • Some laptops have inverters — turn DC into AC
  • Verify backlight: look closely, use a flashlight
  • May need to replace LCD inverter or display — choose carefully
3.1
Display Attributes
Pixel Density
  • Pixels per inch (PPI) — apply a physical measurement to a display
  • Higher pixel density looks better — higher clarity and sharpness
  • 27" 4K display — 160 PPI; 65" 4K TV — 67 PPI
Screen Resolution
  • Number of pixels — width × height
  • More pixels = more detailed display output
  • 16:9 aspect ratio is common; many standards available
Refresh Rate
  • Measured in Hertz (Hz) — frames per second (FPS)
  • Movies: 24 fps; Television: 30 fps; Video games: 60 fps+
  • HDMI 2.1: 4K @ 144 Hz; DisplayPort 2.1: dual 4K @ 144 Hz
Color Gamut
  • Range of colors available on a display — less than human eye capability
  • Standards: sRGB, Adobe RGB, ITU standards for HDTV
  • Higher percentages (coverage) tend to be more expensive
  • OLED provides wide color gamut — best-in-class support
3.2
Network Cables
Twisted Pair Copper Cabling
  • Balanced pair — two wires with equal and opposite signals
  • The twist keeps a single wire constantly moving away from interference
  • Pairs in the same cable have different twist rates
  • Cable categories (Cat5e, Cat6A, etc.) defined by IEEE 802.3
  • Minimum Cat5 for 1000BASE-T
Coaxial Cables
  • Two or more forms sharing a common axis — inner conductor and outer shield
  • RG-6 — used in television/digital cable and high-speed internet over cable
UTP vs STP
  • UTP — Unshielded Twisted Pair: most common, no additional shielding
  • STP — Shielded Twisted Pair: additional shielding against interference
  • Abbreviations: U=Unshielded, S=Braided, F=Foil
  • S/FTP = braided overall + foil around pairs
  • Direct burial STP — waterproof, gel-filled, can go underground without conduit

  • Traditional cable jacket: Polyvinyl chloride (PVC)
  • Plenum-rated: Fluorinated Ethylene Polymer (FEP) or low-smoke PVC
  • Required in air-handling spaces (plenum areas) for fire safety
  • May not be as flexible — different bend radius
  • Worst-case planning — important for any structure

3.3
An Overview of Memory
RAM Basics
  • Random Access Memory (RAM) — most common type
  • NOT referring to hard drive or SSD storage
  • Data and programs can only be used when moved to RAM
  • DIMM — Dual Inline Memory Module, 64-bit data width
  • SO-DIMM — Small Outline DIMM, ~half width, used in laptops/mobile
DRAM & SDRAM
  • DRAM — Dynamic RAM: needs constant refreshing or data disappears
  • Random access — any storage location can be accessed directly
  • SDRAM — Synchronous DRAM: synchronized with common system clock
  • Queue one process while waiting for another
DDR Memory Generations
Version Max per DIMM Notes
DDR3 16 GB Twice data rate of DDR2, no backward compat
DDR4 64 GB Faster frequencies, no backward compat
DDR5 64 GB Faster transfers, key has moved, no backward compat
3.3
Memory Technologies
Parity and ECC
  • Parity memory — adds a parity bit, won't always detect or correct errors
  • ECC — Error Correcting Code: detects AND corrects errors on the fly
  • ECC used in VM servers, database servers, critical systems
  • Looks the same as non-ECC memory externally
Multi-Channel Memory
  • Two independent memory buses — doubles the throughput
  • Dual-channel, triple-channel, or quad-channel
  • Memory combinations should match — exact matches are best
  • Memory module slots are often colored differently to indicate channels
  • Measured in MT/s (Megatransfers per second)
3.4
Storage Devices
HDD & SSD
  • HDD — non-volatile magnetic, spinning platters, random access, moving parts
  • Mechanical components limit speed and can break
  • SSD — non-volatile memory, no moving parts, very fast performance
M.2 Interface
  • Smaller form factor — no SATA data or power cables
  • Can use PCIe bus — 4 GB/s+ throughput with NVMe PCIe x4
  • Different key types: B key, M key, or B+M key
  • M.2 doesn't guarantee NVMe — may use AHCI, check documentation
NVMe vs SATA
  • SATA uses AHCI — designed for hard drives, up to 600 MB/s
  • NVMe — Non-volatile Memory Express, designed for SSD speeds
  • Lower latency, higher throughput, uses PCIe bus
  • Often connects via M.2 interface
SAS / mSATA / Flash / Optical
  • SAS — Serial Attached SCSI: SATA can do 6 Gbit/s, SAS up to 22.5 Gbit/s
  • mSATA — Mini-SATA: quickly replaced by M.2
  • Flash drives — EEPROM, non-volatile, limited writes, not for archival
  • Optical — CD-ROM, DVD-ROM, Blu-ray; slow, archival, internal and external

3.4
RAID
RAID Overview
  • Redundant Array of Independent Disks — RAID is NOT backup
  • RAID 0 — Striping (no redundancy)
  • RAID 1 — Mirroring (high redundancy)
  • RAID 5 — Striping with one parity drive (min. 3 disks)
  • RAID 6 — Striping with two parity drives (min. 4 disks, survives 2 failures)
  • RAID 1+0 (RAID 10) — stripe of mirrors (min. 4 disks)
RAID Comparison
Level Min Disks Redundancy Performance
RAID 0 2 None (zero) Highest write speed
RAID 1 2 High Read fast, write slower
RAID 5 3 1 disk failure Good balance
RAID 6 4 2 disk failures Parity overhead
RAID 10 4 Mirror level Speed of stripe
RAID 0 Note: Zero redundancy — one drive failure breaks the entire array.
RAID 5: Parity rotates across all disks — efficient, one-failure tolerant.
RAID 10: Speed of striping + redundancy of mirroring = the best of both worlds.
3.5
Motherboard Form Factors
ATX Form Factor
  • Advanced Technology Extended — standardized by Intel in 1995
  • Power: 20-pin (original) or 24-pin connector + additional 4/8-pin
  • The standard full-size motherboard
microATX (uATX, M-ATX)
  • Smaller than ATX with limited expansion slots
  • Backward compatible — similar mounting points and power
  • Very popular — many different available options
ITX Form Factor
  • Series of low-power motherboards by VIA Technologies (2001)
  • Mini-ITX is screw-compatible with ATX — fits almost any enclosure
  • Small form factor uses — single-purpose computing (streaming media, thin clients)
Choosing a Motherboard
  • ATX — full featured, media/video editing, gaming, high-end workstations
  • microATX — good balance of features and size
  • Mini-ITX — thin clients, media centers, low-power devices
3.5
Motherboard Expansion Slots
PCI (Legacy)
  • Peripheral Component Interconnect — created in 1994
  • 32-bit and 64-bit bus width, parallel communication
  • Common on previous computer generations — PCIe is the newer tech
PCI Express (PCIe)
  • Replaces older PCI standard — communicates serially (unidirectional lanes)
  • Lane sizes: x1, x2, x4, x8, x16, x32 ("by 4," "by 16")
  • Slower devices don't slow down everyone else
  • Full-duplex — simultaneous bidirectional communication
3.5
Motherboard Connections
24-Pin Motherboard Power
  • Main motherboard power — provides +3.3V, +/-5V, +/-12V
  • 20-pin was original ATX; 24-pin added for PCI Express power
  • Can connect 24-pin to 20-pin motherboard (some cables are 20+4 pin)
PCIe Power & Headers
  • PCIe 6-pin = 75 watts (+12V)
  • PCIe 8-pin = 150 watts (+12V)
  • Commonly seen on high-power components — video adapters
  • Some cables have 6+2 pin — works on both 6-pin and 8-pin connectors
3.5
Motherboard Compatibility
Intel vs AMD
  • Two solid CPU manufacturers — subtle differences
  • AMD tends to be slightly less expensive
  • Different sockets — motherboard designed for a particular CPU
  • Historically: AMD = value, Intel = performance (these switch places often)
Server Motherboards
  • Multisocket — supports multiple physical CPU packages
  • 4+ memory module slots — many expansion slots
  • Larger ATX-sized — designed for rack-mounted systems
3.5
The BIOS
BIOS Overview
  • Basic Input/Output System — firmware (ROM or flash memory)
  • Initializes CPU and memory — builds the workspace
  • POST — Power-On Self-Test
  • Looks for boot loader to start the operating system
Legacy BIOS
  • Original/traditional BIOS — 25+ years old
  • Older OS talked to hardware through BIOS
  • Limited hardware support — no drivers for modern network/video/storage
UEFI BIOS
  • Unified Extensible Firmware Interface — based on Intel's EFI
  • Defined standard implemented by manufacturers
  • Designed to replace legacy BIOS — graphical and text-based
  • Needed for modern computers
3.5
BIOS Settings
Accessing BIOS
  • Secret button(s): Del, F1, F2, Ctrl-S, Ctrl-Alt-S
  • Windows 10/11 Fast Startup can prevent BIOS access
  • Hold Shift when clicking Restart, or via Settings/Recovery
  • Or interrupt normal boot three times
Boot Options & USB
  • Disable hardware — unavailable to the operating system
  • Modify boot order: USB drive, SSD, hard drive
  • USB permissions — security challenge (very large capacities)
  • DoD banned USB flash media for 15 months in 2008 (SillyFDC worm)
Secure Boot
  • Part of the UEFI specification — digitally sign known-good software
  • Software won't run without proper signature — cryptographically secure
  • UEFI protections: manufacturer's public key, BIOS update signature check
  • Checks OS bootloader digital signature at boot time
BIOS Password & Other Settings
  • BIOS/User Password — system won't start without it
  • Supervisor Password — restricts BIOS changes
  • CMOS — stores BIOS configuration (usually flash memory, battery-backed)
  • Reset with a jumper on the motherboard
  • Virtualization support — Intel VT or AMD-V, added to processor
  • Fan control — motherboard includes integrated fan controller and temp sensors
3.5
HSM and TPM
TPM — Trusted Platform Module
  • Specification for cryptographic functions — hardware for single-device encryption
  • Cryptographic processor — random number generator, key generators
  • Persistent memory — unique keys burned in during production
  • Password protected — no dictionary attacks
  • Contains unique secret key — not available outside this device
  • Roots of trust — can detect if computer has been tampered with
  • Can't use TPM key on another computer
HSM — Hardware Security Module
  • Used in large environments — server clusters, many diverse devices
  • Key backup — secured storage for servers
  • High-end cryptographic hardware — plug-in card or separate hardware device
  • Cryptographic accelerators — perform functions for other devices
  • Only the HSM knows the key
TPM vs HSM: TPM is for a single system (local device, often built into motherboard). HSM is used by many systems across multiple devices (often deployed as a high-end server or appliance in a data center to protect the CA key).
3.5
CPU Features
32-bit vs 64-bit
  • 32-bit — 2³² = ~4.29 billion values, supports up to 4 GB RAM
  • 64-bit — 2⁶⁴ = 18.4 quintillion values, up to 17 billion GB
  • 32-bit OS cannot run 64-bit apps; 64-bit OS CAN run 32-bit apps
  • Windows: 32-bit apps in \Program Files (x86); 64-bit in \Program Files
ARM vs x86
  • ARM — developed by Arm Ltd., simplified instruction set (RISC)
  • Efficient, fast, less power, less heat
  • Traditionally mobile/IoT; now Apple and others use it for desktops
  • x86 — Intel/AMD traditional architecture for desktops/servers
Processor Cores
  • Dual-core / Quad-core / Octa-core / Multi-core
  • Each core has its own cache
  • The entire chip may have a shared cache
3.5
Expansion Cards
Types of Expansion Cards
  • Sound card — high-end audio, home theater, Dolby decoding, multiple inputs
  • Video card — discrete GPU (not part of CPU), separate interface, high performance
  • Capture card — video as input (live streaming, external cameras), high bandwidth
  • NIC — Ethernet connection, additional connections for servers/routers
Installation Process
  • Check motherboard documentation — number and type of slots
  • Check adapter card documentation — minimum requirements, known issues
  • Check web for latest drivers before installing
  • Uninstall previous drivers via Windows Device Manager
  • Verify with Device Manager after installation
3.5
Cooling
Fans & Airflow
  • Cool air pulled through the PC — always check for good airflow
  • Motherboard layout and component location are key
  • On-board fans — designed to cool an entire adapter card (common on GPUs)
  • Standard sizes: 80 mm, 120 mm, 200 mm, etc.
Heat Sinks & Thermal Paste
  • Heat sink — dissipates heat via thermal conduction (copper or aluminum alloy)
  • Fins/grid increase surface area — heat transfers to cooler air
  • They get HOT — don't touch!
  • Thermal paste — place between heat sink and chip, pea-sized application
  • Thermal pad — easy to use, won't leak, not reusable (remove and replace)
Fanless & Liquid Cooling
  • Fanless/passive — no fans, silent, specialized functions (video servers, set-top boxes)
  • Low-power components, carefully measured thermal tests
  • Liquid cooling — coolant circulated through computer, not a new concept
  • High-end gaming, graphics, overclocking
3.6
Computer Power
⚠️ Safety Warning
  • ALWAYS disconnect from power source when working on a device
  • Some devices store charge in capacitors — know how to discharge first
  • Never connect your body to any part of an electrical system
  • Do NOT connect yourself to the ground wire
Power Supply
  • Computer uses DC voltage — most sources provide AC
  • Converts 120V AC or 240V AC to 3.3V, 5V, and 12V DC
  • Dual-voltage: US/Canada 120V 60Hz; Europe 230V 50Hz
  • Auto voltage or manual switch — don't plug 120V supply into 230V!
Voltage Rails
Voltage Used For
+12V PCIe adapters, HDD motors, cooling fans, modern components
+5V Some motherboard components (moving to 3.3V)
+3.3V M.2 slots, RAM slots, motherboard logic circuits
+5VSB Standby voltage
-12V Integrated LAN, older serial ports, some PCI cards
Sizing & Efficiency
  • Power supplies rated by watts — bigger isn't necessarily better
  • 50% capacity rule — PSU runs efficiently with room to grow
  • Video adapters are usually the largest power draw
  • Efficiency range: 80%–96% — less heat, save money
  • Redundant PSUs — hot-swappable, each handles 100% load (runs at 50%)
3.7
Multifunction Devices
MFD Components
  • Printer, Scanner, Fax, Network connection, Phone line connection
  • A lot of things can go wrong — you're going to fix them
  • PCL — Printer Command Language (Hewlett-Packard, widely used)
  • PostScript — Adobe Systems, popular with high-end printers
  • Make sure drivers match the printer language!
Printer Drivers & Firmware
  • Specific to printer model — 32-bit and 64-bit drivers are very different
  • Firmware — internal OS of the device (starts system, connects to network)
  • Update firmware to fix incompatibilities and bugs
Connectivity & Sharing
  • Wired: USB Type B → printer, USB Type A → computer; or Ethernet RJ45
  • Wireless: Bluetooth (limited range), 802.11 infrastructure, 802.11 ad hoc
  • Printer share — printer connected to computer which shares it
  • Print server — print directly to printer, jobs queued and managed on printer
Configuration & Security
  • Duplex — print both sides, save paper
  • Orientation — portrait vs. landscape (printer compensates)
  • Tray settings — plain paper, letterhead, etc.
  • Printer security: user auth, badging, audit logs, secured prints (passcode)
Scanners & Network Scan Services
  • Flatbed scanner — standalone or AIO, may include ADF (multi-page)
  • Scan to email, scan to folder (SMB), scan to cloud (Google Drive, Dropbox)
3.8
Laser Printers
Laser Printer Operation
  • Combines laser, high voltage, charged ions, powdered ink, heat, and paper
  • Very high quality, fast printing speeds, very complex with many moving parts
  • Requires on-printer memory, messy inside
Laser Printer Maintenance Kit
  • All those moving parts wear out — heat and pressure
  • Kit contains: replacement feed rollers, new fuser unit, etc.
  • Check printer's page counter to determine when to perform maintenance
  • Fuser units are HOT — power down first!
  • Reset the page counter when done
Replacing Toner Cartridge
  • "Low" doesn't mean empty — check messages
  • Toner can contain the OPC drum (Organic Photoconductor) — sensitive to light
  • Power down the printer, remove packing strips
Calibration & Cleaning
  • Calibration — adjust density for different toner cartridges
  • Cleaning: water or isopropyl alcohol (IPA) — no harsh chemicals!
  • Outside — damp cloth; Inside — wipe dust away
  • Don't use normal vacuum or compressed air inside
  • Rollers — clean with IPA; skin exposure — wash with cold water
3.8
Inkjet Printers
Inkjet Overview
  • Relatively inexpensive, quiet, high-resolution
  • Expensive ink (proprietary), eventually fades, clogs easily
  • CMYK — Cyan, Magenta, Yellow, Key (Black)
Components
  • Ink cartridge — places drops of ink from CMYK cartridges
  • Print head — some integrated into cartridge; others separate
  • Feed rollers — pick up and feed paper, must be clean and not worn
  • Duplexing — print on both sides (included with some printers)
Inkjet Maintenance
  • Clean print heads — clogged heads cause streaks or missing color
  • Many printers auto-clean daily; can start manually
  • Replacing cartridges — usually separate CMYK, takes seconds
  • Recycle empty cartridges — all plastic
  • Calibration — align nozzles to paper, check print dialog
  • Clearing jams — remove tray paper, gentle firm pressure, check for scraps
3.8
Thermal Printers
Thermal Printer Overview
  • White paper turns black when heated — no ink!
  • VERY quiet — almost silent
  • Paper sensitive to light and heat (and clear tape!)
  • Full-length heating element — no moving print head
  • Common: cash registers, credit card terminals, quiet areas
Thermal Printer Maintenance
  • Paper replacement — different sizes for different printers (keep a list)
  • Impossible to substitute — thermally coated paper only
  • Cleaning heating element — isopropyl alcohol (IPA), cleaning pen, cleaning card
  • Removing debris — blow out (outside), damp cloth, avoid vacuum
  • Paper sensitivity to heat — avoid hot areas, receipts fade over time
3.8
Impact Printers
Dot-Matrix Printers
  • Print head with small matrix of pins — presses against ribbon to mark paper
  • Good for carbon/multiple copies, low cost per page
  • NOISY, poor graphics, niche use cases
  • Tractor feed — holes on sides of paper, continuous feed with perforations
  • Multipart paper — carbonless copying, multiple copies in a single pass
Impact Printer Maintenance
  • Printer ribbon — one long circle, replace when ink becomes too light
  • Modular replacement in less than a minute
  • Print head replacement — takes a lot of abuse, gets hot (watch fingers)
  • Look for a release lever or bar
  • Replacing paper — must feed perfectly into holes, forms positioned correctly
4.1
Virtualization Concepts
Virtualization Overview
  • One computer, many operating systems — macOS, Windows, Linux all at once!
  • Separate OS, independent CPU/memory/network — but really one computer
  • Host-based: your normal desktop plus others
  • Enterprise: standalone server hosting VMs
  • Around since 1967 — IBM mainframe virtualization
Sandboxing
  • Isolated testing environment — no connection to real world/production
  • Virtualize development process — try code, break code, nobody gets hurt
  • Roll back to a previous snapshot
  • Dev → Test → Production pipeline with separate VMs
Use Cases
  • Legacy software — run different app versions in separate VMs
  • Application only runs on previous OS — create VM with older OS
  • Cross-platform virtualization — run Windows/macOS/Linux simultaneously
  • Move between each OS seamlessly, no rebooting
  • Save time and resources — one physical computer
4.1
Virtualization Services
The Hypervisor
  • Virtual Machine Manager — manages virtual platform and guest OSes
  • May require CPU that supports virtualization (improves performance)
  • Manages hardware: CPU, networking, security
Hypervisor Types
  • Type 1 (Bare Metal) — hypervisor IS the primary OS
  • VMware ESXi, Microsoft Hyper-V, Xen Project
  • Type 2 (Hosted) — hypervisor runs in an existing OS
  • VMware Workstation, Oracle VirtualBox, Parallels Desktop
  • VMs run on top of the current OS
Resource Requirements
  • CPU: Intel VT or AMD-V support
  • Memory — above and beyond host OS requirements
  • Disk space — each guest OS has its own image
  • Network — configurable per guest OS via virtual switch
  • Network modes: Shared (NAT), Bridged, Private (no external)
Security Concerns
  • VM escaping — malware recognizes virtual environment, compromises hypervisor
  • Malware can jump from one guest OS to another
  • Guest OS security — use traditional controls (host firewall, anti-virus)
  • Watch for rogue VMs — bad guys try to install their own system
  • VDI — Virtual Desktop Infrastructure: apps run on remote server
  • Desktop as a Service (DaaS), minimal local OS, big network requirement
4.2
Cloud Models
Cloud Computing
  • More than a server hosted elsewhere — manages applications and services
  • Effectively unlimited resources (pay for what you use)
  • Deploy infrastructure in minutes, international scope
Cloud Deployment Models
  • Private — your own virtualized local data center, dedicated resources
  • Public — available to everyone over the Internet, shared infrastructure
  • Hybrid — mix of public and private
  • Community — organizations share the same resources
Service Models
  • IaaS — Infrastructure as a Service (Hardware as a Service)
  • Outsource equipment; you manage and secure it; web server providers
  • PaaS — Platform as a Service
  • No servers/software; someone else handles the platform, you develop
  • Put building blocks together (SalesForce.com)
  • SaaS — Software as a Service
  • On-demand software, no local installation (Google Mail, Microsoft 365)
  • Complete application offering — no development work required
4.2
Cloud Characteristics
Shared Resources
  • Internal/private cloud — dedicated resources, build your own, pay up front
  • External/public cloud — share resources, owned by third-party
  • Metered — pay for what you use (upload, storage, download costs)
  • Non-metered — pay for a block of storage, no per-use costs
Cloud Characteristics
  • Elasticity — seamlessly scale up and down as needed
  • Availability — always available, redundancy provides availability
  • File synchronization — information duplicated across cloud locations
  • Multitenancy — many different clients use the same cloud infrastructure
5.1
Troubleshooting Hardware
POST and Boot Issues
  • POST — Power-On Self-Test: tests major system components
  • Failures noted with beeps and/or codes — check documentation
  • Blank screen on boot — listen for beeps, bad video/RAM/CPU/BIOS
  • BIOS time and settings — bad battery, replace it
  • Attempts to boot incorrect device — set boot order in BIOS
Crash Screens (BSOD)
  • Windows Stop Error / Blue Screen of Death — contains important info
  • Also written to Event Log
  • Startup/shutdown BSOD — bad hardware, bad drivers, bad application
  • Fixes: Last Known Good, System Restore, Rollback Driver, Safe Mode
  • Reseat or remove hardware; run hardware diagnostics
No Power / Blank Screen
  • No power — check source, check PSU (use multimeter)
  • Fans spin but no POST — bad motherboard?
  • Blank screen — check cables (power and signal), input selection, brightness
  • Swap the monitor — try on another computer
  • No video after Windows loads — use VGA mode (F8)
Sluggish Performance
  • Task Manager — check high CPU utilization and I/O
  • Windows Update — latest patches and drivers
  • Disk space — check available space and defrag
  • Laptops — may use power-saving mode (throttles CPU)
  • Anti-virus and anti-malware scan
Overheating & Random Shutdown
  • Overheating — check fans, airflow, heat sinks (clean and clear)
  • Verify with monitoring software (HWMonitor, BIOS built-in)
  • Smoke/burning smell — always disconnect power, replace all damaged components
  • Random shutdown — heat-related, check fans/heat sinks; check Event Viewer
Other Issues
  • Application crashes — check Event Log, Reliability Monitor, reinstall app
  • Unusual noises — rattling (loose), scraping (HDD), clicking (fan), pop (capacitor)
  • Inaccurate date/time — bad motherboard battery (button-style), replace it
  • Bad battery will reset BIOS config on every boot
5.2
Troubleshooting Storage Devices
Failure Symptoms
  • Read/write failure — "Cannot read from the source disk"
  • Slow performance — constant LED activity (retry loop)
  • Loud clicking/grinding — "click of death" — may also include scraping
Troubleshooting Steps
  • Get a backup FIRST — a bad drive is bad
  • Check for loose or damaged cables
  • Check for overheating — especially after startup
  • Check power supply — especially if new devices were added
  • Run hard drive diagnostics from manufacturer
Boot Failure Symptoms
  • Drive not recognized / Boot Device Not Found
  • Operating system not found — drive there but Windows is not
  • Fix: check cables, check BIOS boot sequence, check for disabled storage interfaces
  • Check for removable disks (especially USB), try drive in another computer
S.M.A.R.T. Analysis
  • Self-Monitoring, Analysis, and Reporting Technology
  • Schedule disk checks — built into most drive arrays
  • Monitor metrics over time — watch for changes or incrementing values
  • Automated notifications (email, text, console)
  • Resolve BEFORE it becomes a problem — backup and replace the bad drive
RAID Failure & IOPS
  • RAID failure — almost always very obvious: errors, email notifications, alarms
  • Don't pull drives until you check the console!
  • Data loss/corruption — repairs are difficult and expensive
  • ALWAYS HAVE A BACKUP
  • IOPS — Input/Output Operations Per Second: HDD ~200, SSD ~1,000,000
5.3
Troubleshooting Display Issues
Incorrect Input / No Signal
  • Check both power and signal cable — is it actually connected?
  • Input selection on monitor — HDMI, DVI, VGA, etc.
  • Image is dim — check brightness controls
  • Swap the monitor — try on another computer
Image Quality Issues
  • Fuzzy image — video settings must match display's native resolution
  • Burn-in — LCDs have "image sticking", display white screen for extended period
  • Dead pixels — always black, manufacturing defect, replace the monitor
  • Flickering — check cable pins, check refresh rate and resolution settings
Color & Audio Issues
  • Incorrect color — check monitor tint settings, driver color config, OS night light
  • Audio — check monitor volume controls, verify HDMI/DisplayPort audio output in OS
  • Dim image — check brightness/contrast, auto-dimming, driver settings, backlight failure
Distorted / Sizing Issues
  • Distorted image — hardware failure (LCD panels easy to break), bad cable, bad GPU
  • Software configuration: bad video driver
  • Sizing issues — check display scaling option in monitor settings
  • LCD projector bulbs — operate at ~1,000°C, replace burnt-out bulbs, clean dust filters
5.4
Troubleshooting Mobile Devices
Battery Issues
  • Poor battery health — replace aging battery, disable unnecessary features
  • Check app battery usage: iOS Settings/Battery; Android Settings/Battery
  • Swollen battery — buildup of gas, significant fire risk
  • DO NOT open battery, stop using immediately, dispose properly
Screen & Charging Issues
  • Broken screen — backup first!, replace screen, use clear tape until repaired
  • Improper charging — check cables, remove obstructions from port
  • Don't use frayed or damaged cables, verify power adapter with multimeter
  • Physically damaged ports — not modular, requires system board replacement
Connectivity & Overheating
  • Poor/no connectivity — location matters! Cellular: signal strength, outdoor may be better
  • Wi-Fi: limited range, interference, may need channel/frequency change
  • Overheating — phone auto-shuts down, check app usage, avoid direct sunlight
Liquid Damage
  • Many phones have a Liquid Contact Indicator (LCI) — turns color when wet
  • Power DOWN — don't power it back on!
  • Remove case, cards, back, and battery (if possible)
  • Rice doesn't work well — silica gel is better
  • Wait at least a day — DO NOT charge, press keys, heat, or move
Digitizer & App Issues
  • Touchscreen not responding — iOS: press power, slide off, press power; or hold power+Home 10s
  • Cursor drift — use touch calibration app
  • Can't install apps — check storage space, network connectivity, OS compatibility
  • Degraded performance — update OS/apps, free storage, close background apps
Malware & Stylus
  • Malware — unusual apps, large data transfers, pop-ups, high CPU, excess battery
  • Try a security app or scanner
  • Stylus not working — check battery charge, Bluetooth connection, tip replacement
  • Restart the device — resets OS, network, and everything else
5.5
Troubleshooting Networks
No Network Connectivity
  • Check link light — is it plugged in?
  • Ping loopback (127.0.0.1) — is protocol stack working?
  • Ping local IP — checks local config, adapter, link signal
  • Ping default gateway — connectivity on local network
  • Ping 8.8.8.8 or 9.9.9.9 — outside router connectivity
Limited/No Connectivity (Windows)
  • "Limited or No connectivity" / "No Internet Access" alert in system tray
  • Check local IP — APIPA address (169.254.x.x) = only local connectivity
  • If DHCP address obtained, perform ping tests
Intermittent Wireless
  • Interference — something else using our frequency
  • Signal strength — transmitting power, antenna placement
  • Incorrect channel — usually automatic, look for manual tuning
  • Bounce/latency — multipath interference, flat surfaces
  • Access point placement — locate close to users
Slow Network / Latency / Jitter
  • Slow network — validate with a speed test, evaluate each hop
  • High latency — examine response times at every step, packet captures
  • Jitter — time between frames; excessive jitter causes choppy voice calls
  • Poor VoIP — high speed and low latency required, check internet and local equipment
Port Flapping & External Interference
  • Port flapping — network interface goes up and down repeatedly
  • Check wiring, move between switch interfaces, replace hardware
  • External interference — fluorescent lights, microwave ovens, cordless phones
  • Measure with Signal-to-Noise Ratio (SNR), Performance Monitor
Authentication & Intermittent Internet
  • Authentication issues — verify active session, logout/login, packet capture
  • Intermittent internet — ongoing pings, traceroute, speed tests
  • External issues: work with ISP, have contact/account info ready
  • Check your SLA (Service Level Agreement)
5.6
Troubleshooting Printers
Testing the Printer
  • Print or scan a test page — built into Windows, not the application
  • Use diagnostic tools — web-based utilities, vendor-specific, generic
Bad Output
  • Lines down the page — Inkjet: clean print heads; Laser: scratched OPC drum
  • Faded/blank pages — low toner or ink
  • Double/echo images — OPC drum not cleaned, ghost from previous rotation
  • Garbled print — bad driver, wrong model, wrong page description language
Paper Jams & Queues
  • Paper jam — careful removal, don't rip, don't damage internal components
  • Check tray, pickup rollers, paper weight for creased paper
  • Multiple prints pending — corrupted jobs, print spooler crash
  • Check Event Viewer (Windows-PrintService), monitor queue
Finishing Issues & Orientation
  • Staple jams — each manufacturer has different removal process
  • Incorrect hole punch — verify with driver, may need driver update
  • Incorrect page orientation — check print driver settings and printer console
Tray & Connectivity Issues
  • Tray not recognized — paper size must match tray definition in driver
  • Can't print 14" legal to an 11" letter tray
  • Connectivity — confirm connection type (wired/wireless)
  • Verify IP address config: IP, subnet mask, default gateway, DNS
Grinding Noises
  • Never a good sound — something not operating properly
  • Paper jam, carriage stalled or jammed
  • Check manuals for paper jam removal process
  • May require additional maintenance or printer replacement
