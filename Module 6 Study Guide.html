Part 1 — Internet Connection Types & Hardware ▼
How the Internet is structured

The Internet backbone uses high-bandwidth fiber links connecting Internet Exchange Points (IXPs). ISPs connect their networks at IXPs via transit and peering arrangements. Customers connect through the ISP's nearest Point of Presence (PoP), typically a local telephone exchange.

The connection between a home/office and the ISP's PoP is the WAN interface — a point-to-point link. A digital modem establishes this physical connection, while a router handles IP forwarding between networks.
DSL (Digital Subscriber Line)

Uses higher frequencies on existing copper telephone lines (the PSTN / "last mile").

  • ADSL — Asymmetric. Fast downlink (up to ~24 Mbps), slow uplink (1.4–3.3 Mbps). Common in homes.
  • SDSL — Symmetric. Same uplink and downlink. Better for businesses.
  • VDSL/VDSL2 — Used with FTTC. Faster but shorter range. Over 300m: 52 Mbps down / 16 Mbps up asymmetric; 26 Mbps both ways symmetric. VDSL2Vplus: 300 Mbps down / 100 Mbps up at 250m.
DSL modems are NOT interchangeable. An ADSL modem likely won't support VDSL. A splitter (filter) separates voice and data signals at each phone socket. Connections: RJ11 → phone line; RJ45 → router.
Cable (DOCSIS)

Available through cable TV (CATV) service. Network is Hybrid Fiber Coax (HFC) — fiber core with coaxial last mile. Uses F-type threaded connectors (do not overtighten). DOCSIS supports up to 42.88 Mbps down / 30.72 Mbps up. DOCSIS 4 supports up to 10 Gbps down / 6 Gbps up.

The CMTS (Cable Modem Termination System) aggregates traffic from premises and forwards it via fiber backbone to the ISP.

Fiber (FTTP / FTTH)

Fiber all the way to the premises. Implemented as a Passive Optical Network (PON). An OLT (Optical Line Terminal) sends light through a splitter to each subscriber's ONT (Optical Network Terminal). The ONT converts optical to electrical signal and connects to the router via RJ45.

Satellite
  • GEO (Geostationary) — High orbit, wide coverage, but high latency (600–800 ms RTT vs. 10–20 ms for DSL). Typical speeds: 2–6 Mbps up / 30 Mbps down. Uses VSAT dish (pointed south in northern hemisphere) and DVB-S modem.
  • LEO (Low Earth Orbit) — Lower latency (25–60 ms RTT), better bandwidth (5–220+ Mbps). Satellites move, so antennas use phased array technology to track them. Flat panel antennas used by newer providers.
RTT = Round Trip Time — two-way latency (probe sent + response received). GEO's high latency makes it unsuitable for real-time apps (VoIP, gaming, video calls).
WISP (Wireless ISP)

Ground-based fixed wireless. Directional antenna bridges customer and provider. Lower latency than satellite but requires unobstructed line of sight. Unlicensed bands risk interference. Affected by weather and solar flares.

IPv4 Addressing

A 32-bit address written in dotted decimal notation (4 octets, 0–255 each). Example: 192.168.0.1. Range: 0.0.0.0 to 255.255.255.255.

A subnet mask (or prefix) identifies the network vs. host portion. Example: /24 = 255.255.255.0. With a /24 mask, the first 3 octets are the network ID and the last octet is the host ID.

Private addresses (RFC 1918) cannot route over the public Internet. NAT or a proxy server is required for Internet access from private ranges.
Key host configuration parameters
  • IP address — Unique within the network. First and last addresses in each subnet are reserved (network ID and broadcast).
  • Subnet mask — Distinguishes network portion from host portion.
  • Default gateway — Router IP for reaching other networks. Must be in the same subnet as the host.
  • DNS server(s) — Resolves names to IPs. Typically the same as gateway on SOHO networks.
APIPA (Automatic Private IP Addressing)

When a host is set to use DHCP but cannot reach a DHCP server, Windows assigns an APIPA address from 169.254.0.1 – 169.254.255.254. The host can only communicate with other APIPA hosts on the same segment — no Internet access, no gateway routing. Other OS vendors call this "link-local."

IPv6 Addressing

128-bit address written in hexadecimal, 8 groups of 16 bits separated by colons. Leading zeros in a group can be dropped; one consecutive run of all-zero groups can be replaced by ::.

Example: 2001:db8::abc:0:def0:1234

  • First 64 bits = network ID; last 64 bits = interface ID. No subnet mask needed.
  • Global address — Starts with 2 or 3. Public, Internet-routable.
  • Link-local address — Starts with fe80::. Local segment only.
  • SLAAC — StateLess Address Auto Configuration. Hosts auto-configure using router advertisements. No default gateway needed.
  • Dual stack — Host runs both IPv4 and IPv6 simultaneously. Defaults to IPv6, falls back to IPv4.

Part 4 — DHCP, DNS, VLANs & VPNs ▼
DHCP in depth

DHCP uses UDP (server port 67, client port 68). The DORA process:

  • Discover — Client broadcasts DHCPDISCOVER on the network
  • Offer — Server responds with available IP address and configuration
  • Request — Client broadcasts acceptance of the offer
  • Acknowledge — Server confirms the lease is active

Scope — The range of IP addresses the server can assign. Must exclude statically assigned addresses (e.g., the router's own LAN IP).

Lease — IP addresses are time-limited. Clients renew leases before expiry. If renewal fails, the client must restart DORA and release the old address.

Reservation — DHCP server assigns a specific IP to a specific MAC address every time. Useful for printers, servers, and network devices that need consistent addresses.

DNS in depth

DNS translates FQDNs (fully-qualified domain names) to IP addresses. Hierarchy: root (.) → TLD (.com, .org) → domain → subdomain → host. There are 13 root servers labeled A through M. Client-side "stub resolvers" check local cache first, then query the configured DNS server over port 53.

Email spam management (TXT records)
  • SPF — Lists servers authorized to send email for the domain. Only one SPF record allowed per domain.
  • DKIM — Uses a public key (published as a TXT record) to cryptographically sign email and validate the source server.
  • DMARC — Enforces SPF and/or DKIM. Specifies what to do on failure (flag, quarantine, reject) and enables reporting back to the sender.
VLANs

A VLAN logically divides a physical switch into multiple broadcast domains using VLAN IDs (2–4094) on managed switch ports. VLAN 1 is the default. Hosts in different VLANs cannot communicate directly — they require an IP router. Each VLAN gets its own IP subnet, DHCP scope, and DNS services.

Benefits: reduced broadcast traffic, security segmentation, traffic type isolation (e.g., isolating voice VLAN for QoS prioritization over data VLANs).
VPNs

A VPN creates an encrypted tunnel over a public network (Internet) so remote hosts can securely access a private LAN. A VPN gateway authenticates users and manages the tunnel. Once connected, the remote host appears to be on the local network, limited only by available Internet bandwidth.

VPNs can also be used for site-to-site links (branch office to head office) and as additional security within a local network.

Common exam traps ▼
  • DNS uses both TCP and UDP on port 53 — do not say just one.
  • DHCP uses UDP, not TCP — broadcast is required and TCP does not support broadcast.
  • APIPA address means the host cannot reach the DHCP server — check the DHCP service first when troubleshooting.
  • An ADSL modem will not support VDSL — DSL modems are NOT interchangeable.
  • The default gateway must be in the same subnet as the host.
  • Hosts in different VLANs cannot communicate without a router, even if connected to the same physical switch.
  • GEO satellite has very high latency — bad for real-time applications (VoIP, gaming, video conferencing).
  • 5G mmWave bands are fast but cannot penetrate walls and have very short range.
  • SPF — only one per domain. DKIM uses a public key in DNS. DMARC enforces both and adds reporting.
  • The first and last IP in any subnet are reserved — never assign them to hosts.
  • IPv4 is 32 bits; IPv6 is 128 bits.
  • NAT converts private to public addresses at the router. Hosts with private IPs cannot directly access the Internet without NAT.
  • VLAN 1 is the default VLAN on managed switches — custom VLANs use IDs 2–4094.
